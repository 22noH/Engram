# 로컬 무게이트 — 설계

날짜: 2026-07-19
상태: 승인됨 (사용자: "채팅 서버가 붙었을 때만 로그인 화면이 있어야지")

## 1. 목적

혼자 자기 PC에서 쓰는 동안에는 계정·로그인 화면("내 서버 만들기")이 아예 나오지 않는다.
로그인은 **계정이 존재하는 서버**(팀/원격 시나리오)에 붙을 때만. 16a의 "로그인 필수"를
로컬 단독 사용에 한해 해제하는 사용자 결정.

## 2. 설계

### 2.1 서버 (auth-http + self.adapter)

- `/auth/status` 응답에 `localFree: boolean` 추가 — **계정 0개(미설정) AND 요청 출처가
  루프백(127.0.0.1/::1)**일 때 true. 계정이 하나라도 생기면 false(현행 게이트 복귀).
- ws 연결: **미설정 서버 + 루프백 소켓**이면 인증 프레임 없이 통과 — 기존 brain 모드
  (무인증)와 동일한 권한 경로를 재사용한다(새 권한 클래스 발명 금지). 판정은 요청 시점
  accounts.count() 재조회(첫 계정 생성 즉시 게이트 재가동 — 캐시 금지).
- 원격(비루프백) 접속은 현행 그대로: 미설정=셋업 폼(1회용 코드 필요)·설정됨=로그인.
  setup-code 흐름·계정 API 무변경.

### 2.2 렌더러 (App 게이트)

- `/auth/status`가 `localFree: true`면 게이트 생략(brain 모드의 null 처리와 같은 결).
- 그 외 동작 무변경(configured=로그인 폼, 미설정+원격=셋업 폼).

### 2.3 CSS 버그 (같이 수정)

- `input[type=text]`에만 적용되던 입력칸 스타일을 `input[type=password]`에도 —
  게이트 폼의 아이디/비밀번호 칸 모양이 달랐던 버그.

## 3. 엣지

- 로컬 무게이트로 만든 채널의 creatorId는 기존 brain 모드 관성 그대로(무인증 정체성).
  나중에 계정을 만들면 그 시점부터 로그인 요구 — 기존 채널은 owner(첫 계정)가
  channels.manage로 관리 가능(16b 관성).
- 헤드리스/스모크의 ENGRAM_CHAT_ROLE=brain 경로는 무변경(그 상위 개념 그대로).

## 4. 테스트

- auth-http: 루프백+미설정=localFree true·계정 생기면 false·비루프백=항상 false.
- self.adapter: 미설정+루프백 ws가 무인증으로 채널 API 사용 가능·계정 생성 후엔 거부·
  비루프백은 미설정이어도 인증 요구(기존 테스트 회귀 0).
- 렌더러: localFree=true면 게이트 미표시·configured=true면 로그인 폼(회귀).
- CSS: 시각 확인(재설치 후).

## 5. 비범위

- 로컬 사용자용 "계정 만들기" 설정 진입점(원격에서 setup-code로 만드는 현행 경로로 충분) ·
  OIDC/SSO 변경 · 기존 계정 서버의 동작 변경.
