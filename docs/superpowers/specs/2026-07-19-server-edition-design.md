# 서버 에디션(헤드리스 데몬 + 웹 콘솔 + CLI) — 설계

날짜: 2026-07-19
상태: 방향 승인됨 (사용자: 서버는 CLI와 GUI 둘 다 — 어디 설치돼도 관리자가 편하게. GUI는
웹으로 다른 컴퓨터에서도 접속해 설정 변경. 클라이언트는 로그인 화면 하나로 충분=완료됨).
구현 착수 전 플랜 단계에서 세부 확정.

## 1. 제품 구조 (배포 형태 분리 설계의 후속 — 2026-07-19-local-no-gate-design.md §1)

- **데스크톱 앱** = 스탠드얼론(무게이트) + preset 클라이언트(그 서버 로그인). 완료됨.
- **서버 에디션** = 이 문서. 헤드리스 데몬 + 웹 관리 콘솔 + CLI.

## 2. 설계

### 2.1 데몬

- GUI 없는 서버 프로세스(기존 dist/src/main.js 코어 재사용). 외부 바인드(설정),
  계정·로그인 게이트 상시(서버니까 당연 — 판단 로직 없음).
- 설치: 윈도우=서비스 등록 설치본 · 리눅스/NAS=단일 명령(npm/도커는 플랜에서 결정).
  설치 직후 웹 콘솔 주소+1회용 셋업 코드 안내 출력.

### 2.2 웹 관리 콘솔 (GUI)

- 서버가 자체 서빙(`/admin`) — **관리자는 아무 컴퓨터에서나 브라우저로 접속**해 설정
  변경(다른 PC에서의 원격 관리가 기본 시나리오). 별도 앱 설치 없음.
- 첫 접속 = 셋업 마법사: 1회용 셋업 코드 입력(설치 출력물) → owner 계정 생성.
- 이후 = owner 로그인 후 대시보드:
  - 멤버: 가입 승인/정지/복구/권한(16b 체계 그대로)
  - 서버 설정: 이름·포트·바인드(공개 범위)·OIDC
  - 채널 관리 · 상태/로그(하트비트·최근 로그)
  - **클라이언트 배포 파일(preset.json) 생성/다운로드** — 팀원에게 줄 클라이언트 설정
- 기존 렌더러 Admin 영역(16a)·멤버 관리 UI를 웹 콘솔로 이관/재사용.

### 2.3 CLI

- 웹 콘솔과 동일 기능의 명령판(SSH·자동화): `engram-server setup|user approve|user
  suspend|config set|preset export|status` 류. 기존 setup-code 흐름 재사용.

### 2.4 보안

- 웹 콘솔은 owner 로그인 필수. 셋업 코드는 1회용(기존 관성). HTTPS/리버스프록시
  가이드는 문서로(자체 TLS는 비범위 후보 — 플랜에서 결정).

## 3. 비범위(현시점)
