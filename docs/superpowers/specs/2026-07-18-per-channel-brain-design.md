# 채널별 두뇌 선택 — 설계

날짜: 2026-07-18
상태: 승인됨 (브레인스토밍 완료)

## 1. 목적

지금은 서버의 모든 채팅방이 기본 두뇌 하나를 쓴다. 방마다 다른 두뇌를 붙인다:
코딩방=claude, 잡담방=로컬 qwen, 번역방=gemma — 방마다 다른 팀원. 마지막 이월 항목
(다중 두뇌 등록 스펙 §5 → 설정 전면 UI화 스펙 → 여기).

## 2. 사용자 결정

- 단위 = **채널별** (채널=대화 기억 단위와 일치). 메시지별 오버라이드는 비범위.
- 범위 = **그 채널의 모든 작업** (채팅 응답·코딩·예약 발사·ambient 등 채널 문맥이
  있는 전부). 채널 무관 작업(새벽 다이제스트·인사이트·회의·위키 병합·judge)은 기본 두뇌 유지.
- UI = **채널 ⋯ 메뉴** (반응 모드 옆 "두뇌" 드롭다운) — 접근 B(설정창)·C(이름 규칙) 기각.

## 3. 설계

### 3.1 데이터 (chat-store)

- `ChatChannel`에 `brain?: string` 추가 (미설정=기본 두뇌=현행). respondMode와 동일 결:
  로드 시 정규화(문자열 아니면 드롭), `setChannelBrain(id, brain: string | null): boolean`
  (null=지정 해제→필드 삭제). 저장은 기존 채널 영속 경로 그대로.
- 두뇌 이름 검증은 ws 계층에서(등록 이름 목록 대조 — 3.3). 이미 저장된 이름이 나중에
  삭제된 경우는 해소 시점 폴백(3.2)이 처리.

### 3.2 서버 해소 (agent-layer)

- **원칙: 이벤트에 실어 나른다.** self.adapter가 멘션/메시지/작업 이벤트를 만들 때
  채널의 `brain`을 `MentionEvent`(류)에 첨부. Discord 어댑터는 미첨부(=기본)=비범위.
- agent-layer.module의 기존 이름→두뇌 캐시(8d 위임기·전문가가 쓰는 resolve —
  프로필별 인스턴스·세마포어 분리 그대로)를 재사용해 **`resolveChannelBrain(name?)`**:
  이름 없음→주입 BRAIN(기본), 이름 있음→resolve(name), **resolve 실패(삭제된 프로필
  등)→기본 두뇌 폴백+warn 로그**(never-throw — 방이 침묵하는 것보다 기본으로 답하는 게 맞음).
- 적용 지점: 채널 문맥이 있는 에이전트 경로 전부 — ReaderAgent(채팅), observe/ambient,
  CodingSpecialist(코딩 위임), 예약 발사 재주입 경로. 정확한 배선 지점은 플랜에서 코드
  확인으로 확정(이벤트가 흐르는 경로를 따라감). 지휘자(8d) 게이트는 해소된 두뇌의
  canDelegate 기준으로 동일 적용(채널 두뇌가 엔그램 하네스면 그 방에서도 위임 가능).

### 3.3 ws 프레임 (self.adapter)

- `setChannelBrain { id, brain: string | null }`: 권한은 setRespondMode와 동일
  게이트(자기 채널 creatorId 또는 channels.manage — 16b 관성). brain이 null 아니면
  **등록 이름 목록에 있는지 검증**(무효면 조용히 무시 — 위반 무시 관성). 성공 시
  channels 브로드캐스트(기존 채널 변경 통지 재사용).
- `brainNames` 요청/응답 프레임(또는 기존 channels 응답에 이름 목록 동봉 — 플랜에서
  기존 프레임 구조 보고 결정): 드롭다운 채우기용, `listBrainNames(configDir)`.

### 3.4 렌더러 UI

- 채널 ⋯ 메뉴: "두뇌" 드롭다운 — 항목 = `기본 (현재: claude)` + 등록 두뇌 이름들.
  선택 즉시 setChannelBrain. 권한 없으면 항목 자체 미표시(기존 ⋯ 게이트 관성).
- 채널 목록: 기본이 아닌 두뇌를 쓰는 채널에 작은 배지(두뇌 이름) — 어느 방이 어느
  팀원인지 한눈에. i18n en 기본+ko.

### 3.5 폴백·엣지

- 채널 두뇌 삭제됨 → 기본 폴백+로그(3.2). UI 드롭다운은 다음 열람 때 "기본"으로 보임
  (채널 brain 필드는 남아 있어도 무해 — 재등록하면 다시 살아남, 명시 해제도 가능).
- brain 모드(로컬 두뇌 서버)도 동일 메커니즘(채널·brains.json 있음). 팀(team) 채널도
  채널이므로 동일.
- 비용 관성: 8d와 동일 — 채널 두뇌가 API면 그 방 사용량은 그 프로필 단가로 집계(기존
  costUsd 경로 무변경).

## 4. 테스트

- chat-store: brain 필드 왕복·정규화·setChannelBrain(해제 포함).
- self.adapter: setChannelBrain 권한 게이트·무효 이름 무시·브로드캐스트·이벤트에 brain 첨부.
- agent-layer: resolveChannelBrain(미지정=기본·지정=resolve·실패=폴백+로그)·ReaderAgent가
  채널 두뇌로 complete 호출·지휘자 게이트가 해소 두뇌 기준.
- 렌더러: ⋯ 드롭다운 렌더·선택 프레임 송신·배지.
- 실스모크: 채널 두 개에 다른 두뇌(기본 claude + 등록된 다른 프로필) 지정 후 각 방
  응답이 해당 두뇌로 가는지(로그/프로필 호출로 확인).

## 5. 비범위

- 메시지별 오버라이드 · Discord 채널 두뇌 · 채널별 모델 파라미터(온도 등) ·
  채널 무관 작업(다이제스트 등)의 두뇌 변경 · 두뇌별 사용량 대시보드.
