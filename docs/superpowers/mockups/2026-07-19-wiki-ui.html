Per-channel brain selection

외부아키텍처2026-07-19 · main f00dc7f

채널 ⋯ 메뉴에서 모델을 고르면 그 방의 모든 작업(채팅 응답·observe·코딩 위임·예약 발사·재시작 재개)이 그 모델로 돈다. 미설정 = 기본 모델 = 기존과 동일(회귀 0).

아키텍처 — "이벤트에 실어 나른다"

  • 데이터: ChatChannel.brain?: string (chat-store). trim 저장, setChannelBrain(id, brain|null) — null이면 필드 삭제(기본 복귀).
  • 전달: self.adapter의 단일 이벤트 생성점에서 brain: ch.brain 첨부 → messenger-bridge → CoreMessage. 예약·재개는 fire 시점 조회(자가치유).
  • 해소: ChannelBrainResolver.resolve(name?) — 기존 이름→모델 캐시 재사용, 요청별 지역 변수로만 해소. 미등록 이름 = 기본 폴백 + warn.

핵심 함정: 지휘자 자기위임 데드락

채널 모델이 지휘자가 되면 위임 목록에 자기 이름이 포함된다 — 같은 인스턴스·같은 세마포어라 concurrency 1에서 영구 데드락. 위임 목록과 런타임 검증 양쪽에서 자기 이름을 제외해 차단.
